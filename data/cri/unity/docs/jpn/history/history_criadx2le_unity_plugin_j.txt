２．動作環境
上記プラグインの動作環境について記述します。

２．１ 動作確認環境
本バージョンに含まれるプラグインとサンプルは次の環境で動作確認済みです。
(1) Unity Editor
  - Unity Ver.4.7.0 および 5.3.2p2

(2) PC
  - Windows 8.1 64bit

(3) Mac OS X
  - Mac OS X 10.10.5
  - Xcode 7.3

(4) iOS
  - iOS 8.0.2
  - iOS 9.3
  - Xcode 7.3

(5) Android
  - Android NDK r10e

２．２ 動作要件
本プラグインを動作させるには、Unityの動作要件に加えて
以下の要件を満たす必要があります。

(1) PC
  - Unity 4 Pro (4.6以降) / Unity 5 (5.0以降)
  - Windows 7 / Windows 8
  - Microsoft .NET Framework 3.5 SP1
  - Microsoft .NET Framework 4.0

(2) Mac OS X
  - Unity 4 Pro (4.6以降) / Unity 5 (5.0以降)
  - Mac OS X 10.6以降

(3) iOS
  - Unity 4 Pro (4.6以降) / Unity 5 (5.0以降)
  - armv7/arm64対応のCPU、iOS5.1.1以降がインストールされた端末
  - シミュレータ上では動作しません。
    (Unityのネイティブプラグインの動作制限のため）

(4) Android
  - Unity 4 Pro (4.6以降) / Unity 5 (5.0以降)
  - Android OS 2.3以降に対応
  - armv7/x86対応のCPU、Android OS 2.3以降がインストールされた端末
    (エミュレータ上での動作確認はしておりません)


３．変更履歴
[2016-03-31 CRI ADX2 LE Unity Plugin Ver.2.17.00] -------------------------
[共通]
(1) 仕様変更
    - REACTのトリガーキューの停止完了を待ってからREACTの戻り処理を開始してい
      たのを、トリガーキューの停止処理開始時にREACTの戻り処理を開始するように
      仕様を変更しました。
      (トリガーキューのリリースとREACTの戻りとでクロスフェードのようなことが
      行えるように仕様を変更しました)
    - .cs、.shaderファイルの一部がUTF8-BOMになっていなかったものをUTF8-BOMに変換
      しました。
    - システム初期化時に作成していたデフォルトカテゴリ機能を廃止しました。
    - ボスリミットグループの発音数が上限に達した状態で新規に同一グループ
      の発音を行う際に空きボイスがあれば、グループ内のボイスを停止後、
      空きボイスを使って発音を行うよう処理を変更しました。
      本修正により、空きボイスがある場合、ボイスリミットグループ上限
      到達時も遅延なく発音が行えるようになりました。
    - CriWareInitializerコンポーネントのACFプロパティに
      Streaming Assets以外のデータを指定できるように変更しました。
      Streaming Assets以外から読み込みを行う場合は、インスペクタではなく、
      スクリプトからパスを指定してください
(2) 機能追加
    - センドレベルの設定APIを追加しました。
      - CriAtomExPlayer::SetSendLevel
    - 音声に同期した再生時刻を取得する関数を CriAtomExPlayback に追加しました。
      - CriAtomExPlayback::GetTimeSyncedWithAudio
    - 上記関数による再生時刻取得を有効にするためのフラグを CriAtomExPlayer の
      初期化パラメータに追加しました。
      - CriAtomExPlayer::CriAtomExPlayer(bool)

    - REACTで曲線形状の変化補間が使用できるように機能を追加しました。
    - 曲線形状のエンベロープが設定された音声データを再生できるようになりました。
    - キューの再生開始後、任意時間同一キューの再生を禁止する機能を実装しました。
      禁止時間の設定はAtomCraftで行ってください。
    - カテゴリ単位でカテゴリを参照しているキューの再生開始後、任意時間同一
      カテゴリ参照キューの再生を禁止する機能を実装しました。
      禁止時間の設定はAtomCraftでカテゴリに対して行ってください。
    - トラック遷移再生でのトラック毎の遷移先別遷移タイプ設定機能に対応しました。
    - AISACのPan Spreadグラフに対応しました。
    - ストリーミングデータの枯渇による音途切れが発生した際、エラーコール
      バック関数に警告（W2015080610）が返されるようになりました。
    - メモリ上にロードしたACFとACBデータをロードするためのAPIを追加しました。
      - CriAtomEx.RegisterAcf(byte[] acfData)
      - CriAtom.AddCueSheet(string name, byte[] acbData,
                                string awbFile, CriFsBinder awbBinder = null)
      - CriAtomExAcb.LoadAcbData(byte[] acbData, CriFsBinder awbBinder,
                                string awbPath)
    - DSPバスのレベル設定APIを追加しました。バス単位でのレベル調整ができます。
      - CriAtomExAsr.SetBusMatrix
    - ブロックシーケンス再生のデータによる遷移先ブロック指定機能に対応しました。
    - シーケンスの再生位置を取得する機能を追加しました。
      - criAtomExPlayback::GetSequencePosition
    - Unity Editor上で操作可能な、CriAtomWindowの改善を行いました。
    - 複数のリスナーを配置し、切り替えるための機能を追加しました。
      - CriAtomListner::ActivateListener
    - CriAtomExクラスのキュー情報構造体に以下のメンバを追加しました。
      - ignore_player_parameter:プレーヤパラメータ無効化フラグ
      - probability            :再生確率
    - CriAtomExAcb に複数のAWBをアタッチする機能を追加しました。
      - CriAtomExAcb.AttachAwbFile
      - CriAtomExAcb.DetachAwbFile
    - CriAtomSource コンポーネントに CriAtomExPlayer を取得するプロパティを
      追加しました。
      - CriAtomSource.player
    - CriAtomコンポーネントに、シーン遷移時の挙動を変更するフラグを追加しました。
      - CriAtom.dontRemoveExistsCueSheet
      シーン遷移前に保持しているキューシートを、遷移後に削除しなくなります。
    - CriWareInitializerコンポーネントに「dontInitializeOnAwake」プロパティを
      追加しました。初期化パラメタをスクリプトで変更したい場合は、本プロパティを
      有効にし、CriWareInitializer.Initialize関数を明示的に呼び出してください。
    - ファイルディスクリプタの節約モードを追加しました。
      - CriWareInitializer.fileSystemConfig.minimizeFileDescriptorUsage
    - CriAtomExPlayerクラスにフェーダ機能に関する以下のメソッドを追加しました。
        - CriAtomExPlayer.AttachFader
        - CriAtomExPlayer.DetachFader
        - CriAtomExPlayer.SetFadeOutTime
        - CriAtomExPlayer.SetFadeInTime
        - CriAtomExPlayer.SetFadeInStartOffset
        - CriAtomExPlayer.SetFadeOutEndDelay
        - CriAtomExPlayer.IsFading
        - CriAtomExPlayer.ResetFaderParameters

    - Unity 5.2以降でプラグインを使用した場合に警告(CS0618)が発生してしまう
      問題を修正しました。
    - CriWareLibraryInitializer のインスペクタ表示が崩れる問題を修正しました。
    - CriAtomSource 生成直後の音声再生時に、不正な3Dポジショニング効果が
      働いてしまう問題を修正しました。
    - CriAtomExAcb::DetachAwbFile 関数内で処理がデッドロックする可能性が
      ある不具合を修正しました。
    - 現在のピッチ以上の値を毎V設定し続けると、いつまで経ってもピッチが
      変更されない不具合を修正しました。
    - 音声再生中にピッチを大幅に上げた場合（1Vで700セント以上げた場合）に、
      音途切れが発生する不具合を修正しました。
      本修正により、上記操作を行った場合でも音途切れが発生しなくなります。
      一方、ピッチが上がるタイミングが前バージョンより1V遅くなっています。
      （ピッチ変更に耐えうるだけのデータ量をバッファリングしてからピッチ
      が変わるよう、動作を変更しています。）
    - プレイリストモード再生時にアクショントラックが初回しか再生されない不具
      合を修正しました。
    - キューリンクを使用した階層構造データで上位階層のMuteが正常動作しない
      不具合を修正しました。
    - ACFが登録されていない状態でACBファイルをロードし、
      CriAtomEx::UnregisterAcf 関数を実行すると、誤ったエラー情報（ACFが
      登録されているかのようなエラー）が返される不具合を修正しました。
    - マルチAWB機能を使用したACBによるストリーム再生で、常に1つ目のAWBを使用
      して再生が行われてしまう不具合を修正しました。
    - アクショントラックが含まれるキューをシーク再生すると、シークによるアク
      ション読み飛ばし時に不正なワーニングが発生する不具合を修正しました。
    - ACFデータ内に存在するラベル情報へのアクセスに失敗する可能性のある不具
      合を修正しました。
    - 分割遷移設定されたブロックシーケンス再生で、遷移時に多重に遷移先ブロッ
      クを再生してしまう可能性のある不具合を修正しました。
    - 複数のカテゴリを参照するキューに「REACT用シーケンスエンド」マーカーを
      設定すると、カテゴリの参照順によってREACTが駆動したままになってしまう
      不具合を修正しました。
    - 音声再生中に以下の関数を実行した場合に、ごくまれに音途切れが発生
      する不具合を修正しました。
      - CriAtomExAcb::LoadAcbFile
    - 音声データのループ区間の長さや、再生時のピッチが特定の値になった
      場合に、ループ位置で音途切れの危険性が高くなるケースがあったため、
      バッファサイズを調整しました。
    - 特定のビルドモジュールをインストールしていない環境で発生していた
      スクリプトのコンパイルエラーを修正しました。
    - プラグインに含まれるコンポーネントに対するインスペクタ上での変更が
      セーブされない不具合を修正しました。
    - Unity 5.3.0以降で非推奨となったAPIの使用を取り止めました。
    - Prefab化したGameObjectをシーンに配置した状態で、CriWareIntializerなど
      のコンポーネントの設定値を変更しても、変更が保存されない不具合を修正
      しました。
    - CriAtomListener の OnEnable より先に、CriAtomSource の OnEnable が
      実行された場合に3Dポジショニングが動作しない場合があった問題を修正
      しました。
    - CriWareInitializerコンポーネントをAddした際、NullReferenceExceptionが発生
      することがある不具合を修正しました。
    - タイミングによって、CriAtomSourceの3Dポジショニング設定が有効にならないケ
      ースがある不具合を修正しました。
    - 再生中のトラック遷移タイプキューに対してパラメータをアクションを使用し
      て変更すると、設定値の２倍値になってしまう不具合を修正しました。
    - 複数のカテゴリに属するキューに対して、カテゴリのポーズが適用
      されない場合があった問題を修正しました。
    - カテゴリに属するキューに対して、カテゴリをポーズ状態にして再生を開始
      した際に、シーケンスの時刻が進んでしまう不具合を修正しました。
    - ブロックシーケンス再生において、データ内に設定されている遷移設定とは
      ことなる遷移をしてしまう可能性がある不具合を修正しました。
    - フェーダをアタッチしたプレーヤに対しcriAtomExPlayer::Stop関数を
      実行すると、再生中にステータスが一瞬Prepに戻る不具合を修正しました。

    - 音源のサンプリングレートよりも高いピッチが音声データに設定されて

    - 再生中のACBハンドルに対しCriAtom.RemoveCueSheet関数を実行した際、
      ごくまれに長時間（30秒程度）処理がブロックされ、"E2011080599"エラー

    - 対応するセレクタ情報がACF内に存在しないセレクタタイプのスイッチキュー

    - パンタイプが3Dポジショニングのときに3D音源設定がなくても、センドレベル

[Android]
(1) 仕様変更
    - 他機種用にエンコードされた音声データを再生できるようになりました。

    - 発音遅延時間を推測する機能を追加しました。
        - CriAtomExLatencyEstimator.InitializeModule()

      アプリケーションが本プラグインに音声の再生を要求してから
      実際にスピーカーが発音するまでの遅延時間を推測します。
      詳細については本プラグインのマニュアルを参照してください。

      これに伴い、Androidのプラグイン配置を変更しました。

        + armeabi-v7a
          - libcri_ware_unity.so
        + x86
          - libcri_ware_unity.so
      従来とプラグイン内のファイル構成が変わるため、アップデートの前に
      古いCRIプラグインを削除してください。

(1) 機能追加
    - スマートフォン上でのインゲームプレビューに対応しました。
      CRI Atom Craft上でターゲット設定を行うとWiFi通信でスマートフォン実機上で
      インゲームプレビューが可能になります。
